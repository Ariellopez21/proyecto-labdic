\section{Introducción}

Con el modelo de base de datos definido en el hito anterior, el presente hito aborda 
la implementación completa del backend del sistema. Partiendo del esquema relacional 
ya establecido, se desarrollan los servicios, repositorios, DTOs y controladores que 
conforman la API REST para los seis módulos funcionales del proyecto: Usuarios, 
Autenticación, Catálogo, Productos, Dispositivos y Préstamos.

El desarrollo del backend no siguió un proceso lineal y estático. A medida que cada 
módulo tomaba forma, surgieron ajustes en el modelo de datos que debieron gestionarse 
mediante migraciones controladas con Alembic, manteniendo la coherencia entre el 
esquema de la base de datos y el código Python. Este proceso iterativo es parte natural 
del desarrollo y queda documentado como parte del hito.

Cada módulo se implementa siguiendo una arquitectura consistente basada en tres capas: 
repositorio, DTO y controlador. Esta uniformidad no solo facilita el mantenimiento del 
código, sino que establece un patrón reproducible que los hitos siguientes podrán consumir de forma predecible a través de una API 
bien definida y documentada.
