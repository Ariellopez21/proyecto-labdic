\section{Objetivos}\label{sec:objetivos_informe_final}

\subsection*{Objetivo general}

Desarrollar un sistema web de gestión de préstamos e inventario para el laboratorio del 
Departamento de Ingeniería en Computación, que digitalice y centralice los procesos de 
administración del inventario y el ciclo de vida de los préstamos, mediante una arquitectura 
cliente-servidor modular con control de acceso diferenciado por roles.

\subsection*{Objetivos específicos}

\begin{itemize}
	\item Establecer las bases conceptuales y tecnológicas del proyecto, identificando las 
	herramientas adecuadas para el sistema y analizando las necesidades del laboratorio 
	que el sistema busca resolver.
	
	\item Diseñar e implementar un modelo de base de datos relacional que sustente la 
	gestión de usuarios, catálogo de referencia, productos, dispositivos y préstamos, 
	garantizando consistencia referencial, escalabilidad y trazabilidad mediante el 
	control de versiones con Alembic.
	
	\item Implementar el backend completo del sistema, desarrollando los módulos funcionales 
	de Usuarios, Autenticación, Catálogo, Productos, Dispositivos y Préstamos mediante 
	servicios, repositorios, DTOs y controladores, exponiendo una API REST documentada 
	que sirva de base para el desarrollo del frontend.
	
	\item Desarrollar el frontend completo del sistema a través de fases progresivas, 
	implementando las vistas, componentes, stores y servicios para que administradores 
	y usuarios interactúen con todos los módulos mediante una interfaz funcional, 
	responsiva y con control de acceso basado en roles.
	
	\item Documentar la arquitectura técnica del sistema, describiendo la separación de 
	responsabilidades entre las capas del backend, el flujo de consumo de la API y los 
	mecanismos de gestión de estado y control de acceso implementados en el frontend.
\end{itemize}
